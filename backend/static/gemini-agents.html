Эта фраза будет первым, что скажет ассистент при звонке. Максимум 200 символов.

Задайте инструкции для вашего ассистента. Это определит его знания, стиль общения и поведение.

Укажите ID Google таблицы, в которую будут записываться все диалоги с ассистентом. Таблица должна быть доступна для редактирования по ссылке.

Инструкция по настройке:

  • Создайте новую таблицу в Google Sheets
  • Сделайте таблицу доступной для редактирования по ссылке (Настройки доступа → Изменить → Все, у кого есть ссылка)
  • Скопируйте ID из URL таблицы (часть между /d/ и следующим символом /) и вставьте выше
  • Система автоматически создаст необходимые заголовки в таблице
  • В таблицу будут записываться: время, запросы пользователя, ответы ассистента и результаты выполнения функций

Функции ассистента

Включите необходимые функции для вашего ассистента:

Информация о функции:

{
  "name": "send_webhook",
  "description": "Отправляет данные на внешний вебхук (например, для n8n)",
  "parameters": {
    "type": "object",
    "properties": {
      "url": {
        "type": "string",
        "description": "URL вебхука для отправки данных"
      },
      "event": {
        "type": "string",
        "description": "Код события (например, 'booking', 'request', 'notification')"
      },
      "payload": {
        "type": "object",
        "description": "Произвольные данные для отправки"
      }
    },
    "required": ["url", "event"]
  }
}

Пример использования в промпте:

Ты должен использовать функцию send_webhook для отправки данных пользователя на сервер.

URL вебхука: https://n8n.example.com/webhook/123

Когда пользователь запрашивает отправку данных, используй функцию send_webhook с этими параметрами:

  • url: указанный выше URL вебхука
  • event: тип события (например, 'booking', 'request', 'notification')
  • payload: объект с релевантными данными

Пример вызова:

{
  "url": "https://n8n.example.com/webhook/123",
  "event": "booking",
  "payload": {
    "name": "Иван Петров",
    "time": "15:00",
    "date": "2023-05-10"
  }
}

Информация о функции:

{
  "name": "search_pinecone",
  "description": "Ищет похожие документы в Pinecone векторной базе данных",
  "parameters": {
    "type": "object",
    "properties": {
      "namespace": {
        "type": "string",
        "description": "Namespace в Pinecone для поиска документов"
      },
      "query": {
        "type": "string",
        "description": "Поисковый запрос для векторного поиска"
      },
      "top_k": {
        "type": "integer",
        "description": "Количество результатов для возврата",
        "default": 3
      }
    },
    "required": ["namespace", "query"]
  }
}

Пример использования в промпте:

Ты должен использовать функцию search_pinecone для поиска релевантной информации в векторной базе данных.

Pinecone namespace: my_namespace

Когда пользователь запрашивает информацию, используй функцию search_pinecone с этими параметрами:

  • namespace: указанный выше namespace
  • query: запрос пользователя или его перефразировка для лучшего поиска
  • top_k: количество результатов (по умолчанию 3)

После получения результатов используй их для формирования более информативного ответа.

Пример вызова:

{
  "namespace": "my_namespace",
  "query": "Как работает векторный поиск?",
  "top_k": 5
}

        "default": "gpt-4o-mini"
      }
    },
    "required": ["prompt"]
  }
}

Пример использования в промпте:

Ты можешь использовать функцию query_llm для делегирования сложных задач текстовой модели ChatGPT.

Используй эту функцию когда:

  • Пользователь просит написать длинный текст (статью, письмо, код)
  • Нужен детальный анализ или исследование
  • Требуется структурированный ответ с множеством деталей
  • Задача требует глубокого reasoning или пошагового решения

Пример вызова:

{
  "prompt": "Напиши подробную статью о преимуществах искусственного интеллекта в медицине",
  "model": "gpt-4o-mini"
}

Важно: После получения ответа от LLM, ты можешь озвучить краткое резюме пользователю, а полный текст будет показан на экране.

Информация о функции:

{
  "name": "read_google_doc",
  "description": "Читает текст из публичного Google Документа по ссылке",
  "parameters": {
    "type": "object",
    "properties": {
      "url": {
        "type": "string",
        "description": "Ссылка на Google Документ (должен быть публичным или доступным по ссылке)"
      }
    },
    "required": ["url"]
  }
}

Пример использования в промпте:

Ты можешь использовать функцию read_google_doc для чтения содержимого Google Документов.

Используй эту функцию когда:

  • Пользователь дает ссылку на Google Document
  • Нужно прочитать инструкцию или документацию из Google Docs
  • Требуется проанализировать содержимое документа
  • Пользователь просит резюмировать или обработать текст из документа

Пример вызова:

{
  "url": "https://docs.google.com/document/d/ABC123XYZ/edit"
}

Важно: Документ должен быть публичным или доступным по ссылке. После получения текста ты можешь отвечать на вопросы о его содержимом.

Информация о функции:

{
  "name": "add_google_sheet_row",
  "description": "Добавляет новую строку в Google Таблицу. Таблица должна быть доступна для редактирования.",
  "parameters": {
    "type": "object",
    "properties": {
      "url": {
        "type": "string",
        "description": "URL Google Таблицы"
      },
      "row_to_append": {
        "type": "string",
        "description": "Данные для добавления, разделенные ;; (например: 'Иван Петров;;ivan@mail.com;;+79991234567')"
      },
      "sheet_name": {
        "type": "string",
        "description": "Название листа (опционально, по умолчанию первый лист)"
      }
    },
    "required": ["url", "row_to_append"]
  }
}

Пример использования в промпте:

Ты можешь использовать функцию add_google_sheet_row для записи данных в Google Таблицы.

Используй эту функцию когда:

  • Пользователь просит записать/сохранить информацию в таблицу
  • Нужно добавить контактные данные, заявку или бронирование
  • Требуется логировать действия пользователя
  • Пользователь хочет оставить feedback или заполнить форму

Пример вызова:

{
  "url": "https://docs.google.com/spreadsheets/d/ABC123XYZ/edit",
  "row_to_append": "Иван Петров;;ivan@mail.com;;+79991234567;;Консультация",
  "sheet_name": "Заявки"
}

Важно:

  • Данные разделяются символами ;; (двойной точкой с запятой)
  • Количество значений должно соответствовать количеству колонок в таблице
  • Таблица должна быть расшарена на сервисный аккаунт с правами редактора
  • Если sheet_name не указан, запись идет на первый лист таблицы

      }
    },
    "required": ["reason"]
  }
}

Пример использования в промпте:

Ты можешь использовать функцию hangup_call для завершения телефонного разговора.

⚠️ Важно: Эта функция работает только при телефонных звонках через Voximplant!

Используй эту функцию когда:

  • Пользователь поблагодарил и завершил разговор
  • Пользователь прямо попросил завершить звонок ("до свидания", "спасибо, все")
  • Разговор достиг естественного завершения
  • Превышено время разговора
  • Возникли технические проблемы

Пример вызова:

{
  "reason": "conversation_completed",
  "farewell_message": "До свидания! Хорошего дня!"
}

Причины завершения (reason):

  • conversation_completed - разговор естественно завершен
  • user_request - по просьбе пользователя
  • time_limit_reached - превышен лимит времени
  • technical_issue - техническая проблема
  • emergency - экстренное завершение

Примеры прощальных сообщений:

  • "До свидания! Хорошего дня!"
  • "Спасибо за обращение! До встречи!"
  • "Всего доброго!"
  • "Обращайтесь еще!"
